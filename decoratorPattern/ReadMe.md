# 装饰器模式

## 角色组成
1. 组件：用于规范接收附加责任的对象
2. 具体组件：实现组件的类，并通过装饰器为其添加一些职责
3. 装饰器：是一个包含具体组件对象的类，可以通过其子类扩展具体组件的功能
4. 具体装饰器：实现装饰器类的相关方法，并且给具体组件对象添加附加职责
5. 客户端：使用多层装饰器封装组件，前提是它能使用通用接口与所有对象交互

## 使用场景
1.  在不修改代码情况下使用对象
2. 能与通用接口交互的对象
3. 继承和扩展对象方案难以实现

## 实战示例
1. 组件：手机获取价格
2. 具体组件：基础零件类，获取价格
3. 装饰器：用于设置具体组件，和获取手机价格
4. 具体装饰器：IPhone手机和Xiaomi手机
5. 客户端：获取不同手机搭配基础零件的价格
